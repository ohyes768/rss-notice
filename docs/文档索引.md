## 5. 文档规范

### 5.1 文档风格

- 使用 Markdown 格式
- 中文撰写
- 代码示例使用语法高亮
- 表格清晰展示字段和参数

### 5.2 版本号规范

遵循语义化版本 (Semantic Versioning):
- **主版本号**: 不兼容的 API 变更
- **次版本号**: 向下兼容的功能性新增
- **修订号**: 向下兼容的问题修正

### 5.3 变更记录规范

每次更新文档时，在相应章节添加变更记录：

```markdown
### YYYY-MM-DD - 版本号 - 变更标题

**新增**:
- ✅ 新增内容1
- ✅ 新增内容2

**修改**:
- 🔧 修改内容1
- 🔧 修改内容2

**删除**:
- ❌ 删除内容1
```
